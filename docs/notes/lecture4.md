## 理解 static：类级别 vs 对象级别

| 特性 | 非静态 (Instance) | 静态 (Static) |
| :--- | :--- | :--- |
| **内存位置** | 存在于每个堆对象中 | 存在于方法区（全局只有一份） |
| **如何调用** | 必须先 `new` 对象，通过对象调 | 直接通过 `类名.名` 调用 |
| **生命周期** | 随对象创建而生，随垃圾回收而死 | 随类加载而生，程序结束才死 |
| **相互访问** | 可以访问静态成员 | **不能**直接访问非静态成员 |

### 核心结论

- **Static** 就像是“全人类”的属性（如：人口总数）。
- **Non-static** 就像是“具体某个人”的属性（如：你的名字）。
- `main` 必须是 `static`，因为它是程序的“点火开关”，必须在没有汽车（对象）的情况下就能发动引擎。
